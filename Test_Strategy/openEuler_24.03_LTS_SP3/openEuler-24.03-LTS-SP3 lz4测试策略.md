关键词： 
RISC-V, LZ4, Zicclsm, 内存访问优化, 性能优化
 
摘要：
本文档描述了针对支持Zicclsm扩展的RISC-V处理器优化LZ4压缩库内存访问方式的特性测试策略。通过启用直接内存访问模式，在RISC-V平台上实现34%的性能提升。


缩略语清单：

| 缩略语 | 英文全名 | 中文解释 |
| ------ | -------- | -------- |
| LZ4 | Lempel-Ziv 4 | 快速无损压缩算法 |
| RISC-V  |  Reduced Instruction Set Computer - V  |  精简指令集计算机第五代 |
| Zicclsm  | RISC-V Misaligned Atomics Extension | RISC-V未对齐访问扩展 |

## 特性应用场景分析
<!-- 主要描述特性的应用场景分析，指导后面场景测试的测试策略制定 -->
1. 数据压缩场景：在RISC-V服务器上进行日志压缩、数据备份等需要高性能压缩的场景
2. 实时传输场景：网络数据传输中的实时压缩，如数据库复制、文件传输等
3. 存储优化场景：文件系统压缩、内存压缩交换等对性能敏感的场景

# 特性测试执行策略

## 特性测试依赖描述
<!-- 主要描述特性测试所依赖的执行环境、软件包、环境变量等依赖 -->
1. RISC-V工具链（GCC >= 14.1）
2. 支持Zicclsm扩展的RISC-V处理器或QEMU模拟器
3. 标准测试数据集（如Canterbury Corpus）

## 特性测试约束
<!-- 主要描述特性测试的约束条件 -->
1. 需要在支持和不支持Zicclsm的环境下分别测试
2. 必须在真实的RISC-V硬件上测试，不建议使用模拟器
3. 性能测试需在相同硬件环境下对比
