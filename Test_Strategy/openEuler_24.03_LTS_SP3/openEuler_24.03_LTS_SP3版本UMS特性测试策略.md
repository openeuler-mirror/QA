# 特性描述
<!-- 主要介绍特性实现的背景、功能以及作用 -->
UMS是一种北向兼容标准socket API，南向基于UB网络进行数据传输，透明加速TCP通信的内核网络协议栈。UMS在SMC-R协议和内核SMC-R源码基础上进行二次开发，基于UMDK实现共享内存通信协议（SMC，Shared Memory Communication），旨在充分发挥华为UB硬件设备的性能优势，提升整体网络传输效率。作为构建在UB网络架构之上的高性能通信协议，UMS提供低延迟、高吞吐量的网络传输能力，特别适用于对网络性能要求苛刻的场景，如Redis、数据库、AI训推、分布式缓存等

## 特性实现流程描述
<!-- 主要描述特性实现的流程，可使用流程图等方式描述 -->
### 建链流程
1. TCP handshake：UMS协议栈为每个UMS socket维护一个内核TCP socket(clcsock)，首先与对端的clcsock进行handshake，确认是否具有UMS能力，如果没有，则UMS回退到TCP。
2. CLC：UMS协议栈创建（或复用）link资源，通过lcsock的CLC消息交互信息，如Jetty id、eid、tseg、token等。
3. LLC：如果是first contact，客户端和服务器还需要通过LLC消息验证link的可用性。

### 数据传输流程
1. sendmsg：发送端UMS协议栈把数据从用户态拷贝到内核态该connection对应的SndBuf。
2. Write RMB：发送端通过URMA Write操作把SndBuf数据写入到对端RMB。
3. Write with IMM notify：发送端通过URMA Write with IMM语义，将通知信息嵌入到数据流中，通知接收端数据写入的位置。接收端收到Write with IMM消息，解析出数据写入位置，然后把数据写入用户态recv buffer。
4. CDC reply：必要时，接收端回复CDC消息，同步状态到发送端。

## 风险项
<!-- 主要描述特性已知风险项 -->
UMS建链无认证，数据传输为明文，存在安全风险。应用在使用UMS时，应对标TCP socket，在应用层开启TLS认证和加密传输，保证端到端通信安全。
