# 1     特性概述

​    cu-concrete安全加固工具是专为企业级打造的安全加固框架，重点解决信创替代场景中批量加固效率低、安全基线适配难、传统加固脚本耦合度高、参数硬编码、运维成本高等问题，能够自动化检测和修复系统层面的安全隐患。工具采用分层模块化设计，整合CLI命令行、TUI可视化和WEB网页三种交互能力，内置 55 项标准化加固项，支持参数化配置，无需修改代码即可适配不同安全基线；开放框架支持按规范自主开发、自动集成新增加固模块，实现安全能力持续演进。同时，支持按云池、按列表批量下发加固任务，实时监控执行状态，支持自动化配置生成和配置脚本微调，任务完成后可查看历史执行日志，形成从任务下发、执行监控到结果查看的全流程可追溯管理。

​    本工具采用配置化、开放式设计理念，所有加固策略通过YAML文件实现参数化管理，无需改动代码即可快速适配不同等级的安全基线要求。同时制定统一的开发与目录规范，支持各团队自主开发新加固模块，新增策略可被框架自动识别集成，具备极强的扩展能力。工具依托文件化轻量化存储实现加固状态持久。该工具可用于信创服务器国产化改造项目，有效提升批量加固效率、缩短合规整改周期、降低人工运维成本，为大规模异构服务器集群提供稳定、合规、可审计的安全防护体系。

# 3     测试结论概述

## 3.1   测试整体结论

本次针对 cu-concrete 特性共计执行 38 条测试用例，覆盖安全加固全流程。测试过程中发现问题均已完成修复并回归验证，所有用例执行通过，无功能性缺陷，工具整体质量良好。


## 3.2   约束说明

- cu-concrete 必须使用 root 权限运行，普通用户无法启动与执行操作；
- 同一主机仅允许运行一个 cu-concrete 进程，重复启动会触发拦截；
- 新增部门 / 新增加固项后，需手动删除`flag.txt`、`data_status.pkl`状态文件，并重新执行全量扫描，否则新增策略无法加载；
- 批量加固前需提前完成 云池配置（config.yaml）、Ansible 主机清单文件配置，确保主机网络连通、SSH 密钥认证正常；
- Docker 相关加固项依赖主机已部署 Docker 服务，无 Docker 环境时对应加固项可跳过执行。
