# 特性描述
远程证明服务可实现分布式系统之间的硬件验证信任。它通过加密证明验证远程节点（例如，云实例）的完整性，确保它们在可信状态下运行真实软件。

Server：主要用于持续验证部署了代理的机器，确保其完整性和可信性。

CLI：用于管理服务所需的输入/输出文件或策略信息。

Agent：部署在需要验证的机器上，收集证据，与服务建立可信关系。

Key_Manager：使用第三方密钥管理工具存储和提供服务在验证过程中所需的加密密钥。

# 特性分层策略
## 总体测试策略
- 测试目标：本次测试主要覆盖功能测试和可靠性测试。功能测试包括：远程证明支持DICE配置文件非法值校验，接口参数非法值校验，agent获取证据正常，server证据校验成功，RBS服务启动，接口正常调用等；可靠性测试包括：动态库加载失败，报文格式错误，等待应答超时等。
- 时间进度：待评估
- 风险评估：NA
- 重点事项：同测试目标
- 争议事项：NA

## 功能测试
| 功能描述 | 设计思路 | 测试重点 | 备注 |
| ------- | ------- | ------- | ---- |
| 安装启动 | 修改agent配置文件，开启DICE插件，服务正常启动 | 服务启动过程日志无报错，后续接口调用正常 |  |
| 配置文件非法值校验 | 对于agent配置文件新增配置项进行非法值校验 | agent配置文件配置项配置非法值，重启agent失败，日志有对应报错记录且可支持定位 | |
| 接口参数非法值校验 | 调用远程证明接口，传入非法值进行校验 | 返回码正确，日志有对应记录且可支持定位 | |
| 端到端功能正常 | 依次调用获取nonce，上传策略，获取证据，证据校验等接口，进行端到端测试 | 远程证明支持DICE端到端功能正常 | |
| 服务管理 | RBS服务支持通过 systemd 管理系统服务的启停与状态监控 | systemctl start/stop/status/restart 命令对RBS服务有效；服务开机自启配置可持久化 | |
| 服务管理 | 调用RBS相关接口，服务正常执行 | RBS服务正常启动，调用相关接口，模块及端到端功能正常 | |

## 可靠性测试
| 测试目标 | 设计思路 | 测试重点 | 备注 |
| ------- | ------- | ------- | ---- |
| 动态库加载失败 | agent、server启动时插件预加载检测 | agent、server端启动时插件预加载检测，动态库不存在时日志记录error信息，提示插件加载失败 | |
| 等待应答超时 | 对应阶段停掉agent、server服务 | 采集时停掉agent服务，验证时停掉server服务，构造故障造成超时报错，检查日志有错误信息 | |
| 报文格式错误 | 调用接口时篡改报文 | 篡改报文，服务端解析失败，检查日志有错误信息 | |
| 长稳测试 | 7*24h，连续端到端全流程远程证明 | 不存在组件崩溃服务异常，内存会随着运行时间慢慢上升，但是不会超过最大占用占比无泄漏现象 | |
