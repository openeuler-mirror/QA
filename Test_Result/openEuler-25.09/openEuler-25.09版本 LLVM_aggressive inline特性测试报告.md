# 1     特性概述

用户在编写代码时，在函数定义时加入 ``__attribute__((noinline)) ``属性，使该函数无法内联进被调用点，一般用户出于调试、压力测试、实验等目的添加该属性，并可能忘记删除。一般情况下内联函数会带来额外的性能收益，并且使编译器能够挖掘更多的优化机会，因此aggressive inline特性能够忽略noinline属性，激进的强制内联相关函数。
