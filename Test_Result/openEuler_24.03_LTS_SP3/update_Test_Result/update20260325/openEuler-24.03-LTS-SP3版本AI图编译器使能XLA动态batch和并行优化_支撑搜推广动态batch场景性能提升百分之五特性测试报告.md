# 1     特性概述

搜推广业务存在使用tensorflow框架的CPU推理场景，并广泛使用动态batch，但原生XLA图编译器只支持静态shape，动态batch场景由于不断触发重编译导致性能劣化严重，无法发挥图编译器的优化能力。
计算图从Tensorflow到XLA需要一个转换过程，通过自动聚类将部分计算子图自动分组成“cluster”， 每个cluster对应一个XLARun。我们基于原有的聚类算法，增加数据并行分析，将计算瓶颈的可并行子图切分为不同的cluster，充分利用inter并行
