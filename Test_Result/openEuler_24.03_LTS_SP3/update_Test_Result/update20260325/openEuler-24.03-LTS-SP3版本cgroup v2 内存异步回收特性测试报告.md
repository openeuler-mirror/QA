# 1     特性概述

互联网厂商大多使用系统容器作为虚拟化方案,即在物理机上启动系统容器,容器间cgroup隔离。存在如下痛点:当某个容器上的节点内存使用较大时,比如发生实例间数据拷贝,单个容器内存耗尽,free查看显示为0,但是整机内存还很富裕,于是内核不触发主动内存回收,该容器卡死。当前cgroup支持同步内存回收，是业务进程来进行回收的，所以对业务性能提升无帮助。
基于以上痛点以及当前情况，本需求基于memory.high支持异步内存回收，通过警戒水位线和安全水位线，减少由于memory.high触发的同步内存回收，从而在保证容易不卡死的前提下，缓解由于由于同步内存回收带来的性能下降。
