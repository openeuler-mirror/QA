# 1     特性概述

A5虚拟机QEMU支持UB mem D2H。在A5虚拟化场景，使用UB总线代替PCIE总线连接NPU卡和CPU，由于当前UB总线不支持NPU以全虚拟化的方式访问虚拟机内部内存（类似于SMMU页表），所以需要提供一种半虚拟化的方式，让位于虚拟机内部的UB驱动主动将KVCache的信息（GPA、SIZE等）传递给Host，然后由Host侧建立UMMU页表，此后NPU就可以通过UB总线访问这些KVCache内存了。

## 3.2   约束说明

1.	NPU直通场景不支持虚拟机热迁移、热替换、休眠唤醒、快照、内存复用等特性。
2.	当前不支持直通NPU卡热插拔特性。
3.	本功能设计Guest和Host的交互，在UB信息从Guest传递到Host侧可能会造成Guest内部的卡队，1GB的Kvcache信息传递会造成约1秒以内的卡顿。
4.	其他性能指标暂未承诺。
5. X86不支持。
