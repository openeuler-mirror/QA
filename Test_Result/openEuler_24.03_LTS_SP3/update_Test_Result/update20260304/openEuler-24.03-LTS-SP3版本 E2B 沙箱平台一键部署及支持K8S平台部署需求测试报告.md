关键词： E2B、沙箱、AI Agent、firecracker

摘要：E2B是一个专为AI应用设计的**开源云端代码执行沙箱平台**，提供安全隔离的容器化环境来运行不受信任的代码（如AI生成的Python/JS代码）,其官方开源版本不支持在arm鲲鹏的openeuler上部署，也不支持使用k8s进行管理部署，本需求主要解决这两个问题。

# 1     特性概述

E2B支持在鲲鹏arm环境上快速完成集群部署，支持使用k8s部署

# 3     测试结论概述

## 3.1   测试整体结论

云原生需求，共计8执行个用例，主要覆盖了功能测试、可靠性测试。整体开发代码量5.3 k，发现问题1个，代码缺陷密度为0.18/kloc，发现问题已解决，回归通过，无遗留风险，整体质量良好。

## 3.2   约束说明

1. 自建Postpres数据库(请使用postgre 使用14或14以上版本)
2. 自建Harbor仓库

3. 自建minio数据库

4. 至少4个节点，作为4个集群，提供给api集群（边缘节点）、build集群（模板构建节点）、default（沙箱业务节点）nomad server集群

5. 仅支持`e2b-2.9.0`及`e2b_code_interpreter-2.4.1`版本，请配套使用

6. host内核及BIOS需开启虚拟化支持

7. runc版本>=1.0.2

8. docker版本>=25.0.3

9. k8s版本：

   Client Version: v1.32.5
   Kustomize Version: v5.5.0
   Server Version: v1.32.5

10. containerd >= v1.7.13

11. 需基于鲲鹏arm硬件

12. 需基于openeuler 6.6及以上内核
