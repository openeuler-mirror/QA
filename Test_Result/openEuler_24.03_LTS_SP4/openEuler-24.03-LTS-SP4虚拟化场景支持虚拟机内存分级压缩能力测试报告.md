## 5.2   测试流程

### 5.2.1 AR20260511002385：Memlink支持大页内存冷热分级

**接口说明：**

Memlink对外提供两个查询能力，通过集成该接口实现查询能力，memlink_example二进制已集成该能力用于自测试。

- 页面分数查询：`./memlink_example -p pid -a hva`
- 页面访问数量查询：`./memlink_example -p pid -c`

```
int QueryPageScore(int pid, uint64_t addr, uint16_t *score);
 * 查询指定进程指定页面地址的热度评分
 * @param pid   QEMU 进程 PID
 * @param addr  虚拟地址（自动对齐到 2MB）
 * @param score [out] 返回的热度评分 0-65535
 * @return 0 成功，-1 失败（pid 不存在或 addr 不在任何追踪范围内）

int QueryAndClearPageAccessedCount(int pid, uint64_t *accessed_count);
 * 查询指定进程的累计页面访问次数，并清零计数器
 * @param pid            QEMU 进程 PID
 * @param accessed_count [out] 返回的累计访问次数
 * @return 0 成功，-1 失败（pid 不存在）
```

**前置条件：**
- 启动memlink，加载etmem_scan.ko
- 页面分数查询测试场景需要在虚拟机里面访问特定的物理地址
- 方法：关闭编译选项CONFIG_STRICT_DEVMEM构建内核，然后替换到虚拟机里面，即可使用/dev/mem访问物理地址
- 在宿主机用gpa2hva工具查看对应的hva，hva作为分数查询的入参

**单numa场景页面分数查询**

1. 启动虚拟机，只有一个numa node
2. 配置大页：`echo 10240 > /sys/devices/system/node/node0/hugepages/hugepages-2048kB/nr_hugepages`
3. 启动虚拟机：`virsh create /Images/lzk/vm/test/poc/xml/super/poc0.xml`
4. 在虚拟机里面用dev/mem访问物理地址
5. 在host侧查询对应的hva：`virsh qemu-monitor-command poc0 --hmp "gpa2hva 0x60000000"`
6. 查询分数：`memlink_example -p pid -a hva`
7. 在虚拟机里面访问同样的物理地址，等待扫描周期后再次查询，分数会进一步升高
8. 不访问后，分数会降低

**多numa场景页面分数查询**

由于memlink执行walk页面是以page range为粒度的，需要进一步测试多个numa node场景下的分数查询。

1. 启动虚拟机，有两个numa node
2. 在虚拟机里面用dev/mem访问物理地址，对应的hva应该分别位于两个numa node
3. 在host侧查询对应的hva
4. 查询分数，此时分数会升高
5. 在虚拟机里面访问同样的物理地址，等待扫描周期后再次查询，分数会进一步升高
6. 不访问后，分数会降低

**多虚拟机多numa场景**

由于本次提交重构了虚拟机管理部分，需要测试虚拟机生命周期管理功能正常使用。

1. 启动两个虚拟机，每个虚拟机有2个numa node
2. 在虚拟机1里面用dev/mem访问物理地址，对应的hva应该分别位于两个numa node
3. 在host侧查询对应的hva
4. 在虚拟机1查询分数，此时分数会升高
5. 在虚拟机1里面访问同样的物理地址，等待扫描周期后再次查询，分数会进一步升高
6. 不访问后，分数会降低
7. 在虚拟机2重复同样的测试操作

**访问页面数量查询**

1. 启动虚拟机
2. 空载状态下查询页面访问个数
3. 在本扫描周期再次查询页面访问个数
4. 给虚拟机加压：`./alloc 1 2`
5. 再次查询页面访问个数

**Libvirt重启场景测试**

1. 启动虚拟机，在虚拟机里面用dev/mem访问物理地址
2. 在host侧查询对应的hva
3. 查询分数，此时分数会升高
4. 在虚拟机里面访问同样的物理地址，等待扫描周期后再次查询，分数会进一步升高
5. 不访问后，分数会降低
6. 查询访问页面个数
7. 重启libvirt再次执行上述查询动作

### 5.2.2 AR20260511002649：内核支持大页内存分级

**基本功能测试**

1. 启动一个超过大页规格的虚拟机
2. 虚拟机内部加压到超过 free 大页
3. 循环加压，运行12h+
4. 测试结果：虚拟机能正常使用，host内核正常，观测到free -h或者swapon --show里面swap空间被使用

**设备直通互斥测试**

1. 配置2G大页，4G交换空间：`echo 1024 > /sys/devices/system/node/node0/hugepages/hugepages-2048kB/nr_hugepages`
2. 启动4G虚拟机
3. 测试结果：虚拟机配置直通设备后，虚拟机直接起不来

**交换空间打满测试**

1. 配置2G大页，4G交换空间
2. 启动一个16G虚拟机
3. 在虚拟机里面加压8G
4. 测试结果：加压host的dmesg会打印一直重试申请内存

**交换磁盘IO压力测试**

1. 配置2G大页，4G交换空间
2. 启动一个8G虚拟机
3. 在虚拟机里面加压4G
4. 磁盘加压：`fio -filename=/dev/sdb -direct=1 -rw=randwrite -bs=1M -size=1G -iodepth=1 -ioengine=libaio -numjobs=1 -group_reporting -name=fio_test -output=/home/fio_test.log -time_based -ramp_time=0 -runtime=99999`
5. 测试结果：磁盘加压，虚拟机正常运行
