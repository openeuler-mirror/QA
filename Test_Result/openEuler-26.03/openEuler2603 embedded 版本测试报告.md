  # 1   概述

  openEuler Embedded是基于openEuler社区面向嵌入式场景的Linux版本，旨在成为一个高质量的以Linux为中心的嵌入式系统软件平台。

  本文主要描述openEuler 26.03 embedded版本的总体测试活动，按照社区开发模式进行运作，结合社区release-manager团队制定的版本计划规划相应的测试计划及活动。测试报告覆盖新需求的测试执行情况和评估，并结合各类专项测试活动和版本问题单总体情况进行整体的说明和质量评估。

  # 2   测试版本说明

  openEuler 26.03 embedded版本是 24.03 LTS SP3 版本的扩展版本，同时支持构建5.10内核和6.6内核镜像。面向嵌入式场景，持续提供更多新特性和功能扩展，给开发者和用户带来全新的体验，服务更多的领域和更多的用户。该版本基于openEuler 24.03 LTS SP3 分支，发布范围相较24.03 LTS SP3版本主要变动：

支持具身智能IB-Robot开发框架，主要包含如下功能：
1) 底层感知: 通过 ROS 2 Driver 统一接入多路相机 (USB/RealSense)、雷达及麦克风。
2) 多样化采集: 支持 VR 手柄、Xbox 控制器及手机 IMU 等遥操作设备，为模仿学习提供专家示范数据。
3) 协议转换枢纽 (tensormsg): 作为架构的枢纽，tensormsg 负责 ros_msg 与 tensor 之间的双向转换，通过合约（Contract）机制保证数据流的类型安全与一致性。
4) 推理与研发服务 (Inference Service): 支持各类 VLA（视觉-语言-动作）大模型（如 SmolVLA, Pi0.5）以及端到端策略模型（如 ACT, Diffusion Policy）。系统支持 自动检测后端 并根据控制模式按需启动。
5) 统一动作执行器 (Action Dispatch): 充当机器人的“小脑”。在 ACT 模式下负责 Action Chunking 调度与高频插值；在规划模式下对接 MoveIt 2 执行受限轨迹，并提供统一的 RobotStatus 汇报。
6) 配置驱动中心 (robot_config): 实现“规格驱动本体行为”。通过单一 YAML 定义关节、控制器模式及传感器外参，支持一键切换仿真与实机环境。

  # 5   问题单统计

  openEuler 26.03 版本共发现问题单27个，其中修复问题单27个，遗留问题单0个，问题单回归均通过。详细分布见下表:

| 测试阶段               | 问题单数 |
| ---------------------- | -------- |
| openEuler embedded RC1 | 5        |
| openEuler embedded RC2 | 3        |
| openEuler embedded RC3 | 19       |
| openEuler embedded RC4 | 0        |

  1、RC1 为操作系统基础测试、外围包、基础命令及安全加固测试，主要聚焦于操作系统的基础功能及基本安全问题，由于嵌入式支持镜像较多，发现问题主要集中在镜像构建、编译、网络异常等。

  2、RC2为操作系统专项测试，针对可靠性、兼容性、安全、zvm、资料等方面进行测试，发现的问题主要为安全相关专项问题，涉及漏洞、安全编译选项、安全配置扫描等方面。

  3、RC3进行了全量测试，主要聚焦于新需求IB-ROBOT的测试，检查了IB-ROBOT是否能正常启动，相关命令是否能达成预期。

  4、RC4进行回归测试。

  5、目前版本问题主要集中在IB-ROBOT测试活动，问题均已及时回归，无新增其他问题，版本具备发布条件。

  本次版本测试在基础功能方面发现少量问题，主要集中在旧有特性编译失败上；而大部分问题则集中在针对IB-ROBOT的测试环节。这反映出新增需求在开发阶段的自测不够充分。
针对当前已发现的质量问题，版本将持续改进：

1. 针对旧有特性：在版本演进和代码合入过程中，建立定期回归机制，对核心旧功能进行功能性验证，确保“老树发新芽”的同时不枯萎，做到问题早发现、早规避，避免堆积到版本测试阶段。

2. 针对新增需求：在制定详细测试策略之前，提前明确开发人员的“自测红线”，要求开发阶段必须完成充分的问题自查，力争将大部分隐患拦截在提测之前，减少版本测试中的问题暴露量。

   通过“新老兼顾、关口前移”，让版本质量更稳、更高效。
